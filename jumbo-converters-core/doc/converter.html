Creating and running JUMBOConverters

Overview

Bugs and tips

  • Running one template directly after another may miss the first line and this means the chunk is missed. Shuffling the order of templates may cure this.
  • Make sure that all regex metacharacters are escaped with a \. The known ones are 
     \ $ ^ * ( ) - [ ] { } ? .
  • the following metacharacters in XML documents must be escaped: 
     & < and possibly '
    to 
     &amp; &lt; &apos;
    (They normally occur in examples)
  • Watch out for (unnatural) floats/integer variation (e.g. 16.0 electrons)
  • Some codes have variants that look very similar but differe by number of spaces, float/integer, commas, etc.
  • Make sure that the template has eaten the full chunk you want to parse. Omitting any record children will show what you have caught.
  • Some floats are very variable (e.g. .100E-06)
  • If endPattern uses the last line, make sure endOffset is set (usually 1)
  • Capitalization: NWChem uses both "Energy Calculation" and "cnergy calculation" as titles for the same basic routine (argh)
  • "Illegal repetition near index". Common reaqson is that JUMBO syntax {X} cannot be used in template pattern.
